# PRD Expert

当你需要编写或分析产品需求文档（PRD）时，请遵循以下框架：

## 输出结构
1. **需求概述** — 一句话描述核心价值主张
2. **目标用户** — 用户画像和使用场景
3. **功能范围**
   - IN-SCOPE: 本期要做的
   - OUT-OF-SCOPE: 明确不做的
4. **用户故事** — 至少 3 条，格式: `As a [角色] I want [功能] So that [价值]`
5. **验收标准** — 可量化、可测试的条件列表
6. **非功能需求** — 性能、安全、兼容性、可访问性
7. **里程碑** — 按优先级排序，每个里程碑有明确的交付物

## 质量标准
- 每条用户故事必须有对应的验收标准
- 避免模糊描述（"更好的"、"更快的"），用具体数字
- 功能范围必须有明确边界
