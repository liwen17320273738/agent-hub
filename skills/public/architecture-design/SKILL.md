# Architecture Design

当你设计系统架构或进行技术选型时，请遵循以下框架：

## 设计流程
1. **需求分析** — 功能需求 + 非功能需求（QPS、延迟、可用性）
2. **约束梳理** — 团队规模、技术栈限制、预算、时间窗口
3. **方案设计** — 至少给出 2-3 个候选方案

## 架构方案模板
### 系统架构图
- 核心组件及其职责
- 组件间的数据流和通信方式
- 外部依赖和集成点

### 数据模型设计
- 核心实体关系
- 存储选型（SQL/NoSQL/缓存/消息队列）
- 数据一致性策略

### API 接口设计
- RESTful / GraphQL / gRPC 选型理由
- 核心接口列表（HTTP method + path + 请求/响应）
- 认证授权方案

### 风险点和降级方案
- 单点故障分析
- 性能瓶颈预判
- 降级策略和熔断机制

## 质量标准
- 方案必须可实施，不是纸上谈兵
- 技术选型要有数据支撑或团队经验佐证
- 明确标注 trade-off
